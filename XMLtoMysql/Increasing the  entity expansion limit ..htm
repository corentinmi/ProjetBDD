TKB00027 Increasing the "entity expansion limit".

Last updated 2008-06-11 18:41:59 UTC by Shaw Terwilliger

Symptom

It may be necessary to increase the "entity expansion limit" set for your Teamprise client when uploading large files or dealing with large source code repositories. If you receive an error message stating "Parser has reached the entity expansion limit "xxxxx" set by the Application", performing the directions listed under resolution should resolve the issue.

This setting does not apply to Teamprise 2.0.

Resolution

The entity expansion limit can be increased by using an additional Java VM argument when launching the Teamprise clients. Use the VM argument -DentityExpansionLimit=XXXX (XXXX is an integer, 1000000 should be sufficient for large projects).

Plug-in for Eclipse

Add your VM argument after the -vmargs argument to Eclipse to change the expansion limit in the Teamprise Plug-in for Eclipse. The command line should now resemble the following (this example is for Windows but the format of the additional parameter is identical on all platforms):

    C:\Program Files\Eclipse\Eclipse.exe -vmargs -Xmx512M -DentityExpansionLimit=1000000

Explorer

To define the entity expansion limit parameter for the Teamprise Explorer, edit with a text editor the TeampriseExplorer.ini file that resides beside the Teamprise Explorer executable. Add a line at the end of the file with contents like -DentityExpansionLimit=1000000, then save your changes to the file. The TeampriseExplorer.ini file will look like the following after edit:

    -vmargs
    -Xms256M
    -Xmx512M
    -DentityExpansionLimit=1000000

Command-Line Client

To define the entity expansion limit parameter for the Teamprise command-line client, edit the launcher script (named "tf.cmd" on Windows and "tf" on Unix) with a text editor. Find the line that begins with "java" or "exec java" and add a string like -DentityExpansionLimit=1000000 after the word "java." For Unix, the line should look like the following after the edit (all one line):

    exec java -DentityExpansionLimit=1000000 -Xmx512M -cp "$tp_classpath" 
    "-Djava.library.path=$base_directory/native" $random_device_property 
    com.teamprise.clc.Main "$@"

For Windows (all one line):

    java -DentityExpansionLimit=1000000 -Xmx512M -cp %TP_CLASSPATH% 
    "-Djava.library.path=%BASE_DIRECTORY%native\win32\x86" com.teamprise.clc.scc.Main %*

Save your changes to the launcher script. The new limit will take effect when you next run the command-line client.
